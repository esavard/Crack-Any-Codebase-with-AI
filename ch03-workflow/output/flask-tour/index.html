The Flask codebase's core architecture revolves around a central Flask application object, managing the lifecycle of HTTP requests and application contexts. Key subsystems include the request/response handling, URL routing and dispatch, application and request context management (globals like request, session, current_app), templating, session handling, and CLI interactions. The selection prioritizes the main entry points, the concrete implementations of the Flask application and Blueprint objects alongside their Sans-IO base classes, and core modules defining how requests are processed, configurations are loaded, and responses are generated within this context-driven framework.
